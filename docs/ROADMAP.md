# Дорожная карта проекта

Проект в статусе «успешный прототип». Дальнейшее развитие — список идей и порядок реализации.

---

## Блок 1: Расширение источников

### 1.1. Текст вместе со ссылкой
Извлекать ссылку из сообщения, даже если рядом есть текст. Текст игнорировать.

### 1.2. Infostart
Добавить парсер для портала infostart.ru — статьи по 1С и интеграциям.

### 1.3. Расширенный парсинг GitHub
Парсить не только README, но и другие .md файлы из репозитория: `docs/`, `ARCHITECTURE.md`, `CONTRIBUTING.md` и т.д.

### 1.4. Arxiv + перевод
Парсинг PDF-статей с arxiv. Краткое изложение статьи. По запросу делается перевод статьи.

### 1.5. Стабилизация моделей
Добавить проверку на доступность модели перед ответом с понятным логированием. Сейчас бот падает если модель недоступна.

## Блок 2: Система хранения

### 2.1. Переход на SQLite
Замена файлового хранения на SQLite. Причины:
- Изоляция данных по пользователям (`user_id`)
- Связи между сущностями (статьи ↔ идеи)
- Подготовка к RAG

Структура:
- `articles` — статьи с метаданными
- `ideas` — пользовательские идеи
- `idea_articles` — связь статей с идеями
- `chunks` — куски текста для RAG

---

## Блок 3: Система знаний

### 3.1. Идеи
Группировка статей по темам. Пользователь создаёт Идею, привязывает к ней статьи. Каждый пользователь видит только свои Идеи.

### 3.2. Парсинг кода из статей
Выделение блоков кода из статей, сохранение с контекстом (заголовок статьи, раздел). Формирование кодовой базы для каждой Идеи.

### 3.3. RAG для Идей
Векторная база для каждой Идеи. Связка: SQLite хранит структуру и метаданные, векторная БД — поисковый индекс по эмбеддингам.

### 3.4. Вопросы к базе знаний
Возможность задавать вопросы по накопленной базе: «Какие подходы к RLHF я читал?», «Что писали про интеграцию 1С с LLM?»

---

## Блок 4: Опционально

### 4.1. Анализ картинок
Анализ изображений из статей через vision-модель. Скриншоты с кодом — распознать и сохранить. Декоративные картинки — игнорировать.

### 4.2. Агент для постов
Формирование поста по статье, отправка на модерацию, публикация в Telegram-канал после согласования.

## Отложено

### Агентность и MCP
Не включено в roadmap как отдельный пункт. Потребность появится естественно при работе с RAG (поиск → анализ → ответ → уточнение). MCP — если понадобится интеграция с внешними инструментами.
